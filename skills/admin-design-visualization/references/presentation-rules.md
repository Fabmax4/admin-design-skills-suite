# 表达方式选型规则

## 指标卡

- 用于首屏概览
- 必须配趋势或解释，不只放大数字

## 进度 / 状态条

- 用于流程、SLA、达成率、阈值线
- 高于复杂图表的优先级

## 对比图

- 用于类别、区域、角色、队列横向比较
- 优先支持排序和异常高亮

## 表格中的可视信号

- 用标签、进度、语义色表达风险和状态
- 当对象仍需逐条处理时，应优先于独立图表
- 关键识别列、状态列、动作列优先保持单行扫读
- 空间不足时优先横向滚动和截断，不优先逐字换行

## 默认优先级

1. 先判断能否用指标卡
2. 再判断能否用状态条
3. 再判断是否必须用对比图
4. 处理型列表优先使用表格中的可视信号
