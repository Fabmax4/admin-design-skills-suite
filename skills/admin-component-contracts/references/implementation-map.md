# 示例实现映射

这份文件记录的是“常见实现形态示例”，用于帮助安装后的 Skill 使用者理解这些模式通常会落在什么位置。

安装到其他项目后，不应假设这些实现名称或目录结构真实存在；应把它们重新映射到你自己的列表页、详情抽屉、统一反馈封装和表格配置实现。

| 抽象模式 | 常见实现形态 | 说明 |
| --- | --- | --- |
| `Summary Metrics` | 指标卡组 / 概览卡 | 首页或工作台的概览包装 |
| `Status Rail` | 阶段轨道 / 状态流程条 | 多阶段流程的状态承接 |
| `Signal Panel` | 风险卡 / 信号卡 / 健康卡 | 风险、波动、健康度等信号包装 |
| `Queue Block` | 队列卡 / 异常分桶卡 | 待处理对象分桶与优先级视图 |
| `Scan Table` | 扫读型表格 | 主列表、执行清单、异常队列等表格实现 |
| `Search Bar` | 关键词搜索栏 | 列表首行搜索入口 |
| `Filter Panel` | 条件筛选区 | 列表筛选条件区 |
| `Pagination Bar` | 分页栏 | 列表底部分页器 |
| `Column Configurator` | 列配置浮层 | 显示列与列顺序配置 |
| `Bulk Action Bar` | 批量动作栏 | 已选数量与批量动作承接 |
| `Rule Block` | 规则摘要块 / 规则说明块 | 规则摘要与适用条件表达 |
| `Snapshot Block` | 对象摘要卡 | 当前对象或当前任务摘要 |
| `Detail Drawer` | 详情抽屉 / 侧边详情板 | 保留父列表上下文的详情承接 |
| `Section Editor` | 分区编辑卡 | 详情编辑区块包装 |
| `Journey Editor` | 步骤编辑器 / 流程编辑器 | 旅程、流程或步骤的维护 |
| `List Editor` | 列表编辑器 | 卖点、规则、摘要项等清单维护 |

## 使用方式

- 如果你自己的当前实现已经满足抽象契约，优先复用
- 如果只是字段差异，优先通过数据结构和插槽扩展
- 只有当抽象契约都不一致时，才考虑新增组件

## 浮层与反馈实现

| 抽象模式 | 常见实现形态 | 说明 |
| --- | --- | --- |
| `Dialog / Modal` | 风险确认弹窗 | 承接阻断式确认或高风险操作确认 |
| `Drawer / Side Panel` | 详情抽屉 / 侧边任务板 | 保留父页面上下文的临时查看或编辑 |
| `Popover` | 锚点浮层 | 承接轻动作或小范围设置 |
| `Tooltip` | 单句说明提示 | 解释字段语义或锁定规则 |
| `Message / Toast` | 轻反馈提示 | 输出短结果、短确认或短告警 |
| `Notification` | 异步反馈通知 | 输出异步完成、系统推送或复杂反馈 |

## 运行时约束

- 全局反馈的程序化入口应收口到项目内统一反馈封装
- 根布局应提供统一反馈上下文
- 如果项目里已有运行时检查，优先通过类似 `check:feedback-runtime` 的命令守住边界
