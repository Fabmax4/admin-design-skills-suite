# 浮层与反馈边界

这份 reference 用来回答一个具体问题：当前任务应该落在 `Modal`、`Drawer`、`Popover`、`Tooltip`，还是 `Message / Notification`。

## 选择顺序

1. 需要先阻断并等待用户决定：`Dialog / Modal`
2. 需要保留父页面上下文，承接临时查看或编辑：`Drawer / Side Panel`
3. 需要锚定在触发器上展示短说明或轻动作：`Popover`
4. 只需要一句解释：`Tooltip`
5. 不需要用户立即决策，只反馈结果：`Message / Notification`

## Dialog / Modal

- 适合：
  删除确认、发布确认、短阻断表单、不可逆操作确认
- 不适合：
  长表单、长详情、需要频繁来回查看父页面的任务
- 关闭规则：
  普通确认可允许 `Esc` 或遮罩关闭；高风险操作默认不允许误触关闭
- 可访问性：
  必须有明确标题，焦点进入浮层后可循环，关闭后回到触发源

## Drawer / Side Panel

- 适合：
  列表中查看详情、编辑子对象、承接不值得跳页的临时任务
- 不适合：
  危险确认、太长的主流程、多层叠加
- 关闭规则：
  必须有清晰关闭语义；存在未保存内容时先确认
- 可访问性：
  必须有标题，关闭后回到来源；不依赖侧栏外的内容才能完成当前最小任务

## Popover

- 适合：
  短解释、状态补充、`1-3` 个轻动作
- 不适合：
  长表单、长文、关键确认、复杂编辑
- 关闭规则：
  点击外部区域或再次触发即可关闭
- 可访问性：
  不能只靠 hover；聚焦或点击也要可触发

## Tooltip

- 适合：
  图标释义、字段解释、按钮文案补充
- 不适合：
  必需信息、段落说明、链接或按钮
- 关闭规则：
  移出或失焦关闭
- 可访问性：
  不能只为鼠标用户存在；键盘聚焦时也要可读

## Message / Toast

- 适合：
  保存成功、刷新完成、轻量错误、批量动作已执行
- 不适合：
  复杂解释、关键下一步决策、长任务状态承接
- 使用上限：
  一次只说一个结果，文案尽量短

## Notification

- 适合：
  异步任务完成、系统主动推送、带标题和补充描述的反馈、需要后续查看入口的提醒
- 不适合：
  高频出现的轻反馈、列表或详情页本该承载的主体信息
- 使用上限：
  只有当 `Message` 装不下、但又不值得弹 `Modal` 时才使用

## 前端落地标准

- 页面和业务组件里统一通过 `useAdminFeedback()` 触发程序化反馈。
- 高风险确认优先 `confirmRisk()`，把误触关闭保护、默认按钮文案和标题结构收口到同一层。
- 短结果优先 `info / success / warning / error`，复杂异步结果优先 `notifyInfo / notifySuccess / notifyWarning / notifyError`。
- 根布局必须包裹 `a-app`，否则不认为反馈上下文完整。
- 禁止在页面里直接散落 `message.*`、`notification.*`、`Modal.confirm()` 或 `AntApp.useApp()`。
