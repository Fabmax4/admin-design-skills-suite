# 抽象组件契约

| 抽象模式 | 最小输入 | 必备状态 | 主动作要求 | 禁止事项 |
| --- | --- | --- | --- | --- |
| `Summary Metrics` | 数值、趋势、说明、动作入口 | 默认、上升、下降、风险 | 至少能进入下一层分析或处理 | 只有数字，没有解释 |
| `Signal Panel` | 当前状态、阈值、影响范围、建议动作 | 健康、关注、告警 | 给出建议动作或恢复动作 | 只报告现状 |
| `Queue Block` | 对象名、责任人、优先级、时限、动作 | 待处理、处理中、已升级、已恢复 | 可直接处理、派发或进入详情 | 只有列表，没有动作 |
| `Scan Table` | 稳定列定义、行对象、主键、状态信号、动作列 | 列顺序稳定、列宽有策略、横向滚动可用、单元格尺寸稳定 | 支持批量浏览、定位、进入动作 | 挤成逐字换行，破坏阅读惯性 |
| `Search Bar` | 关键词范围、占位提示、触发动作、清空动作 | 默认、输入中、已清空、定位结果 | 支持快速定位对象 | 用搜索替代结构化筛选 |
| `Filter Panel` | 筛选项、默认值、重置动作、结果范围说明 | 默认、部分筛选中、已重置、结果收窄 | 支持稳定缩小结果范围 | 隐藏高频筛选、让条件隐形存在 |
| `Pagination Bar` | 当前页、总量、每页数量、翻页动作 | 首页、中间页、末页、空结果 | 支持稳定翻页与回看 | 翻页后丢失查询上下文 |
| `Column Configurator` | 列定义、锁定列、默认顺序、可见状态 | 默认、部分隐藏、重排后稳定、可恢复默认 | 当列数超过 `15` 时支持显示列和列顺序调整 | 允许隐藏主键列或操作列 |
| `Prompt / Notice Block` | 事实、影响范围、时限或阈值、下一步动作 | 默认、提醒、风险、结果反馈 | 帮助用户快速判断和行动 | 只有“尽快处理”“风险较高”这类纯定性表述 |
| `Bulk Action Bar` | 已选数量、动作列表、风险级别、清空选择动作 | 未选中、已选中、执行中、危险确认 | 支持批量处理和二次确认 | 未选中时暴露危险动作 |
| `Rule Block` | 规则标签、规则内容、适用范围 | 默认、警告、限制 | 支持扫读和分组 | 长段堆字 |
| `Snapshot Block` | 对象标题、关键状态、补充字段 | 简洁、稳定、可快速扫读 | 支持查看详情或联动上下文 | 做成完整详情页 |
| `Detail Drawer` | 对象标题、摘要、主任务、关闭语义、返回来源 | 打开、加载中、编辑中、待保存、已回到来源 | 支持在不离开列表的前提下处理临时任务 | 把完整主流程塞进抽屉 |
| `Section Editor` | 区块标题、说明、区块内容、保存语义 | 编辑中、已更新、待校验 | 支持独立编辑与保存语义 | 整页塞进一个巨型表单 |
| `Journey Editor` | 步骤名、责任方、时效、说明 | 顺序明确、步骤可读 | 支持调整步骤内容 | 只有步骤名，没有责任和结果 |
| `List Editor` | 项标题、标签或类型、描述 | 可增删、可排序、可扫读 | 支持快速维护重复项 | 退化成普通段落 |
| `Dialog / Modal` | 标题、触发原因、主次操作、关闭语义 | 打开、确认中、不可逆风险 | 支持明确确认或取消 | 长表单、长滚动、多层嵌套 |
| `Drawer / Side Panel` | 触发对象、上下文摘要、主任务、关闭动作 | 预览、编辑中、加载中、待保存 | 支持回到来源并继续父任务 | 承载危险确认、抽屉套抽屉 |
| `Popover` | 触发器、锚点内容、关闭条件 | 打开、关闭、禁用 | 支持少量轻动作或短说明 | 长文、长表单、关键任务 |
| `Tooltip` | 目标元素、单句解释 | 默认、聚焦或悬停显示 | 不要求主动作 | 承载必需信息、按钮、链接 |
| `Message / Toast` | 反馈级别、短文案、持续时间 | 成功、信息、警告、错误 | 反馈结果即可 | 复杂描述、关键确认 |
| `Notification` | 标题、描述、来源、可选动作 | 信息、异步完成、警告、升级 | 支持查看详情或后续动作 | 高频刷屏、替代主体页面 |

## 可访问性底线

- 交互控件必须可键盘到达
- 状态表达不能只靠颜色
- 标题、说明、操作要能被清晰分离
- 动态反馈不能只靠动效
- 表格在窄视口下优先保行列结构，必要时横向滚动，不优先逐字换行
- `Modal` 和 `Drawer` 关闭后应回到触发源
- `Tooltip` 和 `Popover` 不能只依赖鼠标悬停
- 全局反馈不能成为唯一状态表达方式
- 搜索区、筛选区、分页区应作为一条连续的查询链路工作
- 列配置不应破坏主键列和动作列的稳定位置
- 当列数未超过 `15` 时，优先先靠列宽策略、冻结列和横向滚动控制复杂度
- 批量操作必须先让用户明确自己选中了什么、会影响多少对象
- 提示、播报、提醒、摘要类信息能定量先定量，不能定量再定性
