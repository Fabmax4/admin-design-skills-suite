# 可访问性底线

后台给团队内部长时间使用，可访问性不是加分项，是基础约束。以下五节是**硬底线**，不达到就直接回退，而不是"尽量做"。

## 键盘与焦点

- 键盘必须可达——所有表单、菜单、按钮、表格操作都能用 `Tab` / `Shift+Tab` / `Enter` / `Esc` 走完
- 主内容必须可聚焦——焦点顺序跟视觉顺序一致，不随手设 `tabindex="-1"` 跳过重要元素
- 焦点指示器必须可见，且与背景对比度 ≥ `3:1`——**不允许**用 `outline: none` 把焦点圈关掉而不补回来
- 跳转到新页面或弹层后，焦点要落在合理位置（弹层进入点焦点落在弹层首个可交互元素，弹层关闭后回到触发器）

## 对比度底线

**对比度达不到 AA 的设计直接驳回，这是底线不是建议。**

- 正文文字与背景对比度 ≥ `4.5:1`（WCAG AA）
- 大字（`18pt` / `24px` 以上，或 `14pt` / `18px` bold 以上）≥ `3:1`
- UI 组件边界（按钮、输入框、徽章的边缘）与相邻背景 ≥ `3:1`
- 焦点指示器与背景 ≥ `3:1`
- 禁用态文字可以低一档，但不得低到用户误以为消失了——建议 ≥ `3:1`，并配图标或文案提示"为什么禁用"

验证手段：

- **开发期**：浏览器 DevTools 自带 contrast checker（`Inspect Element` → 悬浮取色）
- **评审期**：装 axe DevTools 或 Lighthouse 扫一遍，跑一遍 `Color Contrast` 规则
- **PR 底线**：推荐在 CI 集成 `@axe-core/playwright` 或 `jest-axe`，把 `color-contrast` 这条当成**必过**规则。`ant-design-vue` 默认主题基本能过 AA，出问题的一般是**自定义覆盖**（改了 `--ant-primary-color` 又没改前景色、或用了浅主色做 filled 按钮的背景）

## 默认态可读性

**所有必要信息必须在默认态就可读，hover 只能是"已可读的东西变得更突出"，绝不能是"从不可见变可见"。**

### 为什么

hover 是鼠标专属——后台的真实用户里有：

- 键盘党（快捷键走流程，鼠标几乎不碰）
- 笔电触控板用户（hover 触发时机不稳）
- 外接触屏 / iPad / Windows 二合一用户（**完全没有 hover 概念**）
- 启用了 `prefers-reduced-motion` 或屏幕阅读器的用户

对他们来说，只有默认态能呈现的信息才是真实存在的信息。

### 硬规则

- 按钮、徽章、链接、状态文字**在默认态就达到 AA 对比度**，不做"淡底淡字 + hover 浮现"
- hover / focus 只做**增强**：背景加深、边框变粗、加下划线、加阴影、加底色——**不承担**"从不可见到可见"的揭示
- 关键信息（订单号、金额、状态、用户名、错误码）不被装饰色块或打码块**整体覆盖**；要打码只打部分字符（`张*三` / `138****1234` / `HC2026041102****`），不做整体遮罩
- 表格里的主键字段（订单号、单据号等）**默认态就必须能读全或能读到足够识别的部分**，需要复制的场景加明确的复制按钮，不靠"鼠标悬浮才显形"

### 典型反例

- 主色 filled 按钮，文字色也改成了主色（或浅主色）——默认态文字几乎跟背景同色，hover 背景加深才"显形"
- 幽灵按钮（ghost/outline）默认态边框和文字都极淡，必须 hover 才能看清
- 表格列用主题色做整块背景覆盖订单号，hover 才露出原文——让用户以为是"保护隐私"，实际是默认态失去信息
- 徽章"淡灰底 + 淡灰字"，hover 才变成"深色底 + 白字"——灰阶对比度不达 `4.5:1`，默认态没法扫读

## 状态表达

- **状态不能只靠颜色**——红色异常、绿色成功必须配图标、文案或形状差异；色盲用户、黑白打印、屏幕滤光下都要能分辨
- 状态徽章用文字作为主要表达（`处理中` / `已完成` / `已拒绝`），颜色只是辅助强化
- 表单校验错误不仅改红色边框，必须在字段下方或气泡里写清**错在哪一个字段、为什么、怎么改**

## 动效降级

- 支持 `prefers-reduced-motion: reduce`——检测到这个偏好就去掉转场、缓动、loading 骨架动画，保留状态本身的变化
- 重要反馈**不能只靠动效**承载——关了动效之后，依然能用文字、图标、状态色读出来
- 页面切换状态应可播报——异步操作完成后 toast 或 aria-live 区域要喊一声，屏幕阅读器才能告诉用户"刚才发生了什么"

## 使用提醒

- 样式决策先过这五节，达不到就回退，不做"先上线再说"
- 后台不是营销页——视觉冲击让位于**可读、可操作、可扫读**
