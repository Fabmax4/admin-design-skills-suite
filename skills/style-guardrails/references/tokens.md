# 样式边界

## 色彩

- 品牌主色：`#fa8c16`
- 成功：沿用 `Ant Design` 绿色语义
- 警告：沿用橙 / 金色语义
- 错误：沿用红色语义
- 信息：沿用蓝色语义

规则：

- 主色只用于关键按钮、重点状态、少量强调
- 语义色只表达状态，不承担装饰任务
- 中性色优先承担结构和阅读任务

## 圆角

- 允许档位：`6 / 8 / 12 / 16`
- 信息卡、表单、表格容器优先 `8` 或 `12`
- 不使用大圆角制造“柔软卡片秀”

## 密度

- 左侧导航展开宽度：`176px`
- 以紧凑、可扫读为目标
- 内边距优先统一，不单独发明节奏

规则：

- 通过分组和版式控制密度
- 不通过把所有字都缩小来换空间

## 间距

- 允许档位：`4 / 8 / 12 / 16 / 24 / 32 / 40 / 48`（对齐 `ant-design` 的 `sizeXXS / XS / SM / MD / LG / XL / XXL` 节奏）
- 表单控件行内最小间距 `8`
- 区块内小元素间距优先 `12-16`
- 区块与区块之间 `16-24`
- 页面主区块之间 `24-32`

规则：

- 不自造 `11px`、`18px`、`22px` 这类中间档
- 强调层级不靠间距翻倍制造"呼吸感"，先靠信息优先级
- 紧凑列表（表格、日志、对账行）可下沉到 `4 / 8` 档，但上下边距仍遵守节奏

## 字号

- 正文：`14px`（对齐 `ant-design` 默认 `fontSize`）
- 次级、辅助文字：`12px`
- 标题档位：`16 / 20 / 24`（对应 `ant-design` 的 `h5 / h4 / h3`；`28 / 30` 仅用于登录/引导等极少数场景）
- 数字、金额等需要连续扫读的字段允许使用 `Tabular Nums`

规则：

- 不使用 `13 / 15 / 17` 这类非档位字号
- 不通过放大字号制造强调，优先用语义色或字重
- 不让正文比 `14px` 更小，`12px` 只留给辅助/标签/时间戳

## 行高

- 正文：`1.5715`（对齐 `ant-design` 默认 `lineHeight`）
- 紧凑表格、日志列表：`1.5`
- 标题：`1.35`

规则：

- 不使用 `1.2` 以下的行高压缩正文
- 表格单元格最小行高 `32px`，中文+英文混排不允许逐字换行
- 标题和段落之间的节奏靠间距而不是压缩行高

## 徽章与标签

后台列表里状态徽章（`Pill / Tag / Badge`）是高频元素，占位必须紧凑，**默认直接用 `ant-design-vue` `<Tag>` 的默认档位**，不要自己再叠 `padding` / `margin`。

档位：

- 高度：`22px`，对齐 `ant-design-vue` `<Tag>` 默认
- 内边距：`0 × 8px`（上下 0、左右 8）
- 圆角：`4px`（比其他容器小一档，徽章本身就是"小元素"）
- 字号：`12px`，不放粗体
- 徽章之间最小间距 `8px`（一行里多个徽章并排时）

规则：

- 不加阴影、不加边框之外的装饰、不做渐变背景
- 不用 `<Button>` 冒充徽章（按钮默认高度 `32px`，比徽章重得多，破坏行密度）
- **一个徽章就是一个状态**，不在徽章下方、右侧或 tooltip 里再叠一行原始英文枚举值（如 `SUCCESS / DEDUCTED`）——状态只出一份，完整说明见 [admin-output-voice](../../admin-output-voice/SKILL.md) 的「页面 UI 文案的额外约束」节
- **徽章文字与徽章底色在默认态就必须达到 WCAG AA 对比度（`4.5:1`）**；`ant-design-vue` `<Tag>` 自带的语义档位（`success` / `warning` / `error` / `processing` / `default`）本身过 AA，**自定义色号时**要用 DevTools / axe 验一遍，不做"淡灰底 + 淡灰字"或"淡主色底 + 主色字"这种肉眼几乎同色的组合
- hover / focus 态只做**增强**：底色加深、边框加粗、文字加粗——**不承担**"默认态看不清、悬浮才显形"的揭示功能（对齐 `references/accessibility-baseline.md` 的「默认态可读性」节）

常见偏航：

- 徽章自加 `padding: 4px 12px` 之类，把一个短短的中文状态裹成一个大盒子，在 `N` 行表格里累积成明显的扫读拖累
- 徽章加 `box-shadow` 或彩色边框（`ant-design-vue` `<Tag>` 只有浅色背景加一层边框就够了）
- 徽章下方再写一行英文码值，等于同一状态展示两次
- 为了凸显状态把徽章做成圆角 `12` 以上的胶囊形，破坏整体圆角一致性
- 为了"更简洁"用淡灰底 + 淡灰字做徽章，hover 才加深——**默认态就是不可读**，等同于要求用户每个徽章都去悬浮一遍
