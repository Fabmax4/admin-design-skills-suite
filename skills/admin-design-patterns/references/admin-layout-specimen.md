# 后台布局样本

UI Kit 和页面模板集需要单独展示后台布局样本，不能只把布局藏在页面截图里。

## 必备样本

| 样本 | 作用 |
| --- | --- |
| 侧栏 | 承载主导航分组和当前页面定位 |
| 顶栏 | 承载全局入口、用户信息、通知和帮助 |
| 面包屑 | 表达当前位置和返回路径 |
| 标签页 | 在同一模块内切换同级任务 |
| 页面头 | 放标题、说明、状态摘要和一级动作 |
| 筛选区 | 放查询条件、重置、保存筛选和结果范围说明 |
| 内容卡 | 承载主要信息或操作区块 |
| 表格卡 | 承载列表、工具栏、批量操作和分页 |
| 侧边信息卡 | 承载公告、帮助、辅助指标或操作建议 |

## 规则

- 布局样本必须说明它解决的寻址、阅读或操作问题。
- 同类页面的导航分组、进入方式、返回路径要保持稳定。
- 页面头只承载标题、状态摘要和一级动作，不堆长文。
- 筛选区与表格区应保持近邻关系，避免用户看不到筛选作用范围。

## 反模式

- 每个页面重新发明导航和返回方式。
- 页面头过大，挤压主要内容。
- 筛选区、表格区和分页区彼此脱节。
- 侧边信息卡喧宾夺主，抢走主任务注意力。
