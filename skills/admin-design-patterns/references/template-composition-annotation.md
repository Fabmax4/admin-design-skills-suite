# 页面模板结构标注

后台页面模板不应只给截图，还要标注每个区域的结构责任。标注让模板可迁移，而不是只可临摹。

## 标注对象

| 区域 | 说明 |
| --- | --- |
| 页面头 | 说明当前页面任务、对象范围和一级动作 |
| 主操作区 | 放最重要的创建、提交、导入、导出或批量动作 |
| 筛选区 / 配置区 | 控制数据范围或配置对象范围 |
| 主内容区 | 承载列表、详情、表单、看板或配置主体 |
| 辅助信息区 | 承载帮助、公告、风险、说明或次要指标 |
| 保存 / 提交区 | 说明保存方式、草稿、发布、取消和返回语义 |

## 规则

- 每个标注都要说明职责，不只写区域名称。
- 标注要能跨业务对象复用，不使用具体行业字段。
- 列表页重点标注页面头、操作区、筛选区、表格区、分页区。
- 编辑页重点标注步骤、分区、校验、保存和发布。
- 详情页重点标注摘要、状态、主动作、历史记录和返回来源。

## 反模式

- 模板只有截图，没有结构说明。
- 标注只描述视觉位置，不解释任务责任。
- 把业务字段名当成模板结构。
- 只标注主内容，不标注操作和返回语义。
