# 页面范式

| 范式 | 核心目标 | 常见主模式块 | 不该做的事 |
| --- | --- | --- | --- |
| 工作台 | 先理解全局，再进入高频动作 | `Summary Metrics`、`Signal Panel`、`Queue Block` | 只堆数字，没有动作入口 |
| 查询列表 | 查找、筛选、比较、批量处理对象 | 筛选区、表格、批量操作区、`Snapshot Block` | 为了信息丰富把详情内容塞进表格 |
| 详情页 | 稳定查看单对象状态、规则、关联信息 | `Snapshot Block`、`Rule Block`、`Signal Panel` | 把详情页做成可编辑大表单 |
| 编辑页 | 非线性维护单对象内容 | `Section Editor`、`Journey Editor`、`List Editor` | 强行改成步骤流 |
| 配置页 | 定义规则、流程、发布控制 | 步骤编排、`Rule Block`、预检区 | 用详情排版承载复杂配置 |
| 异常处理台 | 发现、分桶、派发、恢复异常 | `Queue Block`、`Rule Block`、`Signal Panel` | 只有告警，没有恢复路径 |
| 资源运营台 | 调配资源、容量、处理能力 | `Signal Panel`、计划区、影响范围区 | 只看现状，不给调配动作 |
| 协同台 | 协调多个角色、地点或班次的接手动作 | `Queue Block`、`Status Rail`、`Rule Block` | 只有清单，没有接手与恢复语义 |

## 选择方法

先问：

1. 用户进来是先看全局、先找对象，还是先改内容？
2. 页面主要处理“对象”还是“状态”？
3. 操作是线性流程，还是非线性分区维护？

如果答案偏“看整体”，更接近工作台。
如果答案偏“找对象”，更接近查询列表。
如果答案偏“看一个对象”，更接近详情页。
如果答案偏“改一个对象”，更接近编辑页。
