# 表格操作样本

表格页不是只有列定义。后台表格规范必须把工具栏、筛选、批量操作、行操作、列设置、分页和状态标签作为一个整体样本。

## 必备结构

| 区域 | 作用 |
| --- | --- |
| 主工具栏 | 放新增、导入、导出、刷新、列设置等页面级动作 |
| 筛选入口 | 放关键词、结构化筛选、保存筛选或重置 |
| 批量操作栏 | 只在已选择对象后出现，说明已选数量和影响范围 |
| 表格主体 | 保持列顺序、列宽、固定关键列和横向滚动策略 |
| 行操作 | 放查看、编辑、复制、启停、更多等对象级动作 |
| 状态标签 | 用稳定语义表达对象状态，不叠加多余装饰 |
| 分页 | 保持搜索、筛选、列配置和当前页关系清晰 |

## 规则

- 表格操作区要围绕用户的连续处理路径组织，不要把操作分散到多个无关角落。
- 批量危险动作必须依赖已选对象，并说明影响范围。
- 列数超过 `15` 时必须提供显示列和列顺序配置。
- 空间不足时优先横向滚动和冻结关键列，不把单元格压成逐字换行。
- 分页、筛选、搜索、列配置应共享同一查询上下文。

## 反模式

- 只有表格样式，没有工具栏和批量操作。
- 筛选、分页、列配置彼此割裂。
- 行操作和批量操作语义冲突。
- 状态标签颜色很多，但用户看不出下一步要做什么。
