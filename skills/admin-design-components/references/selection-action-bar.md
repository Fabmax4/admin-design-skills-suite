# Selection Action Bar / 选择操作栏

`Selection Action Bar` 是表格多选后的页面级操作栏。它把“用户选中了哪些对象”转成“可以提交什么动作”，不是普通信息卡，也不是只放按钮的底栏。

## 可识别叫法

- `Selection Action Bar`
- `Selection Summary Bar`
- `Batch Action Bar`
- `Bulk Action Bar`
- `Selected Rows Toolbar`
- 选择操作栏、选择汇总操作栏、已选预览栏、批量操作栏

输出规范时优先使用 `Selection Action Bar`，中文说明可写“选择操作栏”。

## 使用条件

- 表格支持多选。
- 多选后有页面级主动作，例如批量提交、确认、结算、开票、导出、复核、分配、下架。
- 用户需要先看清已选数量、作用范围或关键影响，再执行动作。
- 主动作可能因为金额、状态、权限、跨页选择或风险条件而禁用。

## 最小输入

| 输入 | 说明 |
| --- | --- |
| 已选数量 | 当前页选择数和总选择数要区分 |
| 作用范围 | 当前页、本次筛选结果、跨页全部结果或手动选择集合 |
| 跨页选择语义 | 是否包含未加载行，如何取消跨页选择 |
| 关键金额或影响指标 | 金额、数量、风险数、预计影响对象数等，能定量先定量 |
| 主动作 | 最重要的提交动作，最多保留一个主按钮 |
| 禁用原因 | 主动作不可用时给出可见原因 |
| 清空选择 | 明确回到未选择状态 |

## 必备状态

- 未选择：不显示或以轻提示说明需要先选择对象。
- 已选择：显示已选数量、范围、关键指标、主动作和清空动作。
- 跨页已选：说明是否包含当前筛选条件下的全部对象。
- 执行中：锁定主动作，保留进度或处理中说明。
- 禁用：解释缺少权限、状态不满足、金额不一致、超出范围等原因。
- 危险确认：删除、不可逆提交、财务动作等必须二次确认。

## 布局规则

- 页面级操作栏可贴底或固定到底部，但必须给表格、空态、分页和横向滚动控件预留底部安全空间。
- 优先使用白底、顶部边线和弱上投影；不要做成普通卡片。
- 金额、数量和范围默认保持中性；只有风险、欠额、阻断条件或危险动作才使用语义色。
- 主按钮只放真正的主动作，次要动作收进更多或次级按钮。

## 反模式

- 只显示“批量处理”按钮，不说明选中了什么。
- 把选择操作栏做成表单卡片或摘要卡，混入内容区。
- 贴底后覆盖分页、横向滚动条、表格行或空态。
- 主动作禁用但不说明原因。
- 跨页选择时不告诉用户是否包含未加载数据。
- 多个危险动作平铺在同一操作栏，用户无法判断主次。
