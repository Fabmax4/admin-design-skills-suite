# 弹层决策样本

弹层规范要帮助 AI 判断该用短确认、短表单，还是保留父上下文的抽屉或侧边面板。

## 选择规则

| 类型 | 适合 | 不适合 |
| --- | --- | --- |
| 确认弹窗 | 删除、撤销、发布、不可逆动作确认 | 长说明、复杂表单、长详情 |
| 表单弹窗 | 新建分类、短配置、少量字段录入 | 多步骤编辑、字段很多的维护任务 |
| 抽屉 / 侧边面板 | 列表行详情、侧边编辑、审核、比较 | 必须完全阻断父页面的高风险任务 |
| 浮层提示 | 短说明、轻量选择、列设置 | 必要信息、错误原因、主流程 |

## 必须说明

- 触发对象是什么。
- 是否需要保留父页面上下文。
- 关闭后焦点回到哪里。
- 有未保存内容时怎么处理。
- 是否允许遮罩点击或 `Esc` 关闭。

## 硬边界

- 不把长详情和长编辑塞进 `Modal`。
- 不用 `Tooltip` 承载必要信息。
- 不让多个重量级浮层叠加。
- 不让抽屉关闭后用户丢失列表位置。
