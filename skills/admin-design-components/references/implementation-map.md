# 实现映射

这份文件用于把抽象组件契约映射到目标项目的真实实现。它不假设任何具体代码路径，也不绑定某个前端工程。

## 映射方法

1. 先找现有组件是否已经承担同一抽象模式。
2. 如果只是字段、文案或动作差异，优先通过配置、插槽或组合扩展。
3. 如果状态模型、动作边界或可访问性要求不一致，再考虑新增组件。
4. 新增组件前必须写清：最小输入、状态、主动作、空态、加载态、错误态、禁用态、键盘操作和视觉密度。

## 常见映射

| 抽象模式 | 应映射到的实现位置 | 复用判断 |
| --- | --- | --- |
| `Summary Metrics` | 概览页指标卡或指标组 | 是否只表达核心数量、趋势和阈值 |
| `Status Rail` | 流程状态、任务进度或对象生命周期 | 是否能按固定顺序解释当前状态和下一步 |
| `Signal Panel` | 风险、健康度、异常或建议动作区 | 是否能把信号和后续动作放在同一语境 |
| `Queue Block` | 待办、异常、审批、补救或工单队列 | 是否支持优先级、责任人和处理入口 |
| `Scan Table` | 列表页主表格 | 是否保持列顺序、列宽、固定关键列和连续阅读惯性 |
| `Search Bar` | 列表页关键词搜索 | 是否和筛选、分页共享同一查询状态 |
| `Filter Panel` | 列表页筛选区 | 是否能保存用户常用条件或清楚重置 |
| `Pagination Bar` | 列表页分页器 | 是否和搜索筛选同步更新 |
| `Column Configurator` | 表格列设置入口 | 超过 15 列时是否支持显示列和列顺序 |
| `Bulk Action Bar` | 已选对象后的批量操作区 | 是否说明已选数量、影响范围和危险动作确认 |
| `Selection Action Bar` | 表格多选后的页面级底部操作栏或工具条 | 是否说明已选数量、跨页范围、关键指标、禁用原因，并避免遮挡表格和分页 |
| `Detail Drawer` | 从列表进入的侧边详情 | 是否保留父列表上下文、来源、关闭确认和刷新策略 |
| `Section Editor` | 详情页内分区编辑 | 是否避免把长编辑塞进阻断式弹窗 |

## 浮层与反馈映射

| 抽象模式 | 适用位置 | 不适用位置 |
| --- | --- | --- |
| `Dialog / Modal` | 短任务、阻断确认、高风险提交 | 长表单、长详情、多步骤维护 |
| `Drawer / Side Panel` | 从列表进入详情、编辑、审核、比较 | 必须完全阻断父上下文的任务 |
| `Popover` | 轻量设置、短列表选择、锚点解释 | 承载复杂表单或关键风险说明 |
| `Tooltip` | 补充解释一个词或短字段 | 承载必要信息、错误原因或主动作 |
| `Message / Toast` | 非阻断轻反馈 | 承接需要恢复、确认或追踪的任务 |
| `Notification` | 可延后阅读的系统结果或异步完成 | 替代页面内的任务状态和错误处理 |

## 运行时约束

- 项目应有统一的反馈入口，避免页面到处散落底层反馈调用。
- 根布局应保证全局反馈上下文可用。
- 如果项目已有表格批量操作或底部动作栏实现，应优先扩展为 `Selection Action Bar`，不要再平行新增只服务单页的“已选卡片”。
- 如果项目有运行时或静态检查，应把“统一反馈入口”“不可滥用浮层”“表格列一致性”纳入检查。
