# 组件状态矩阵

可复用后台组件不能只展示默认态。凡是进入 UI Kit 或组件规范集的基础组件，都必须说明关键状态。

## 最小状态

| 组件类型 | 必备状态 |
| --- | --- |
| 按钮 | 默认、悬停、按下、加载、禁用、危险 |
| 输入框 | 默认、聚焦、填充、禁用、错误、只读 |
| 选择器 | 默认、展开、已选、禁用、空选项、错误 |
| 日期选择 | 默认、聚焦、已选、范围选择、禁用 |
| 上传 | 默认、上传中、成功、失败、禁用 |
| 搜索框 | 默认、聚焦、已输入、清空、禁用、无结果 |
| 开关 / 勾选 / 单选 | 未选、已选、禁用、错误或不可用 |
| 状态标签 | 默认、成功、处理中、警告、错误、取消 |

## 判断规则

- 如果状态会影响用户判断或操作，就必须进入矩阵。
- 如果某个状态不会出现，要说明为什么不需要。
- 状态矩阵只描述交互和语义，不绑定某个组件库 API。
- 同类组件状态命名要统一，不要一处叫禁用、一处叫不可用。

## 反模式

- 只有默认态截图，没有聚焦、禁用、错误和加载态。
- 把颜色当状态本身，没有文字或语义说明。
- 只展示组件外观，不说明触发条件和用户下一步。
- 为了填满 UI Kit，人为添加不会出现的状态。
