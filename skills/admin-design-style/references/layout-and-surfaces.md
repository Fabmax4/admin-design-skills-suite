# 布局与表面处理

## 版式

- 优先使用稳定的左右 / 上下分区结构
- 主内容区围绕单一任务组织
- 侧栏、页面头、主内容、辅助信息区层级清晰

## 页面头

- 只承载标题、状态摘要、一级动作
- 不堆长文，不做营销式 Hero

## 边框与阴影

- 优先用边框、背景层次、留白组织区块
- 阴影只做弱化补充
- 不把阴影作为主要分组手段
- 默认使用纯色表面，不用渐变承担主要层级
- 默认信息卡、辅助标签、正文说明不额外染色，先靠结构和排版建立层级

## Selection Action Bar 表面

- 作为页面级操作 chrome 时，优先使用白底、顶部边线和弱上投影，不使用完整卡片边框和大圆角
- 贴底或固定到底部时，内容区必须预留底部安全空间，避免覆盖表格行、空态、分页和横向滚动控件
- 主动作可以使用主色；金额、数量和范围默认保持中性，只有风险、欠额或阻断条件才使用语义色
- 禁用态必须明确变灰，并通过可见文案或可访问说明解释原因

## 常见偏航

- 过大的页面头
- 过重的渐变和背景
- 过多的卡片层叠
- 把普通信息卡、普通文案、普通标签默认染成品牌色或信息色
- 把页面级 `Selection Action Bar` 做成又一张卡片，导致主动作和内容区层级混乱
- 贴底操作栏覆盖分页、表格横向滚动条或空态
- 用情绪化视觉取代结构层级
