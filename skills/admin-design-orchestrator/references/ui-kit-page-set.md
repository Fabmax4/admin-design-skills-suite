# UI Kit 多页规范包

当用户要求设计后台 UI Kit、设计规范包、组件规范集或后台设计系统时，不要把任务降级成单页 demo。默认先规划一套多页标准化交付。

## 触发条件

- 用户明确提到 `UI Kit`、设计规范、组件规范、设计系统、后台规范包。
- 用户给出多张规范图、组件状态图、页面模板图，希望提炼成可复用规则。
- 用户要提高 AI 后台设计能力，而不是只要一张页面效果图。

## 最小页集

1. 基础规范：颜色、字体、间距、圆角、阴影、密度。
2. 导航与布局：侧栏、顶栏、面包屑、标签页、页面头、内容区。
3. 组件状态：按钮、表单、选择器、上传、搜索、状态标签、控件。
4. 数据与表格：指标卡、提示、空状态、消息、表格工具栏、批量操作、分页、列设置。
5. 页面模板：工作台、查询列表、详情页、编辑页、配置页或资金概览等通用后台模板。
6. 弹层与抽屉：短确认、短表单、侧边详情、侧边编辑。
7. 评审清单：用于判断这套 UI Kit 是否能继续指导页面生成。

## 分流链路

推荐链路：

1. `admin-design-principles`：确认 UI Kit 服务的后台类型、使用角色和约束强度。
2. `admin-design-patterns`：规划布局样本和页面模板。
3. `admin-design-components`：补组件状态矩阵、表格操作样本和弹层决策样本。
4. `admin-design-style`：收紧色彩、圆角、密度和表面处理。
5. `admin-design-visualization`：处理指标卡、表格信号、提示和图表边界。
6. `admin-design-review`：按 UI Kit 评审清单验收覆盖面。

## 硬边界

- 不只输出单页大图。
- 不只复刻某张参考图的蓝色视觉。
- 不把行业对象写进规则层；行业对象只能出现在案例层。
- 不把组件库 API 当成规范本身。
- 不在状态、模板、表格、弹层覆盖不足时宣布 UI Kit 完整。
