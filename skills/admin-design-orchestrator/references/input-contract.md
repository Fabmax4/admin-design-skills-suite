# 输入契约

总入口 Skill 接收的任务输入，建议优先归到下面 4 类之一：

## 1. 模糊目标

输入特征：

- 只有业务目标
- 没有明确页面类型
- 没有明确第一优先动作

推荐输入字段：

- `业务目标`
- `目标用户`
- `主要任务`
- `已知约束`

## 2. 页面设计

输入特征：

- 已知页面类型
- 需要细化结构、样式或组件

推荐输入字段：

- `页面类型`
- `第一优先动作`
- `核心信息`
- `辅助信息`
- `已知模式块`

## 3. 页面评审

输入特征：

- 已有方案或已实现页面
- 需要 review、验收或修正

推荐输入字段：

- `审查对象`
- `当前问题`
- `重点担心`
- `是否需要最终验收`

## 4. 组件 / 规范抽象

输入特征：

- 不是做单页，而是定义模式、组件或规范

推荐输入字段：

- `抽象目标`
- `复用范围`
- `已有关联 Skill`
- `希望沉淀的输出`

## 使用规则

- 输入越模糊，越应从上层 Skill 开始
- 输入越具体，越应走最短链路
- 如果用户没有显式提供这些字段，总入口 Skill 应主动从现有上下文里补全最少必要信息
