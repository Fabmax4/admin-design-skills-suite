# 输入契约

总入口 Skill 接收的任务输入，建议优先归到下面 4 类之一。

在归类之前，先判断输入是否足够。如果下面这些关键字段里有多个缺失，总入口应先请求澄清，而不是直接分流：

- `目标用户 / 使用角色`
- `页面类型或任务类型`
- `第一优先动作`
- `成功标准`
- `关键约束`

## 1. 输入信息不足

输入特征：

- 只有业务目标
- 没有明确页面类型
- 没有明确第一优先动作
- 没有明确目标用户或成功标准

推荐输入字段：

- `业务目标`
- `目标用户`
- `主要任务`
- `已知约束`

## 2. 页面设计

输入特征：

- 已知页面类型
- 需要细化结构、样式或组件

推荐输入字段：

- `页面类型`
- `第一优先动作`
- `核心信息`
- `辅助信息`
- `已知模式块`

## 3. 页面评审

输入特征：

- 已有方案或已实现页面
- 需要 review、验收或修正

推荐输入字段：

- `审查对象`
- `当前问题`
- `重点担心`
- `是否需要最终验收`

## 4. 组件 / 规范抽象

输入特征：

- 不是做单页，而是定义模式、组件或规范

推荐输入字段：

- `抽象目标`
- `复用范围`
- `已有关联 Skill`
- `希望沉淀的输出`

## 使用规则

- 输入信息不足时，先澄清，不直接进入 Skill 分流
- 用户在澄清后仍无法准确描述时，再从 `admin-design-principles` 开始
- 输入越具体，越应走最短链路
- 如果用户没有显式提供这些字段，总入口 Skill 应主动从现有上下文里补全最少必要信息
