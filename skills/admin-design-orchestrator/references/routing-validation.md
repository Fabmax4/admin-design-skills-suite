# 路由验收规则

## 文件定位

这是评测层文件，用来判断 `admin-design-orchestrator` 对案例样本的分流是否合格。

- 它服务于案例层验证，不直接充当页面设计规则。
- 页面设计规则仍应维护在原则层、模式层、契约层、样式层等规范文件中。

这份文件用于判断 `admin-design-orchestrator` 对示例调用集的分流是否合格。

## 核心验收项

### 1. 起始层正确

- 是否从预期的第一层开始
- 是否没有越级进入更下层 Skill

### 2. 链路最小化

- 是否只调用必要 Skill
- 是否避免机械跑完整链路

### 3. 上层优先

- 当原则或结构仍不清时，是否避免进入样式、可视化、动效

### 3.5 混合意图处理正确

- 是否识别出主意图和次意图
- 是否先处理主意图，再把次意图延后
- 是否避免把两个链路同时完整跑完

### 3.8 冲突意图处理正确

- 是否识别出硬约束和增强诉求
- 是否优先尊重冻结边界
- 是否避免把互相冲突的两边都当成已满足

### 4. review 时机正确

- 是否只在任务成熟、明确要求 review，或方案已经成型时进入 `admin-design-review`

### 5. 输出可执行

- 输出是否说明：
  - 当前任务阶段
  - 建议调用的 Skill
  - 暂不进入的层
  - 下一步

## 判定方式

### Pass

- 起始 Skill 正确
- Skill 链路和预期接近
- 没有明显越级
- 混合意图时，主意图和次意图排序正确
- 冲突意图时，硬约束优先级处理正确

### Warning

- 起始 Skill 正确
- 但链路偏长，或提前进入了可后置层
- 或混合意图下过早进入了次意图层
- 或冲突意图下没有及时停在边界层

### Fail

- 起始 Skill 错误
- 越过上层直接进入下层
- 没有识别出 review 时机
- 输出无法指导下一步
- 或混合意图下没有识别主意图
- 或冲突意图下假装两个互斥要求都已满足

## 复盘方法

- 如果频繁 `Warning`，优先调整 `workflow-recipes.md`
- 如果频繁 `Fail`，优先调整 `entry-routing.md`
- 如果同一类任务总是分错，说明要补新的样本，而不是立刻改 Skill 逻辑
