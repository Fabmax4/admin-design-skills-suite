# UI Kit 评审清单

评审后台 UI Kit 或设计规范包时，不只看视觉精致度，还要看它是否能指导后续页面生成。

## 覆盖面

- 设计基础是否完整：颜色、字体、间距、圆角、阴影、密度。
- 导航布局是否完整：侧栏、顶栏、面包屑、标签页、页面头、内容区。
- 组件状态是否完整：默认、聚焦、填充、禁用、加载、成功、失败。
- 数据展示是否完整：指标卡、提示、空状态、消息、表格信号。
- 表格系统是否完整：工具栏、筛选、批量操作、行操作、列设置、分页。
- 页面模板是否完整：工作台、查询列表、详情、编辑、配置或其他核心范式。
- 弹层与抽屉是否完整：确认弹窗、短表单弹窗、详情抽屉、侧边编辑。

## 质量判断

- 是否只是在复刻一张视觉图，还是提供了可迁移的结构规则？
- 是否说明每个组件状态的触发条件和用户下一步？
- 是否保持表格行列一致性和连续阅读惯性？
- 是否避免把行业词写成通用规则？
- 是否保持色彩、圆角、阴影和渐变的克制？
- 是否能让另一个 Agent 根据这套 UI Kit 继续生成页面？

## 严重度建议

- `S0`：关键规范缺失，导致 UI Kit 不能指导页面生成。
- `S1`：组件状态、表格操作或页面模板覆盖不足。
- `S2`：布局、色彩、密度或术语有不一致。
- `S3`：示例表达、文案或局部细节可后续打磨。

## 输出要求

评审结论要明确回答：这套 UI Kit 是否已经足以指导后续后台页面生成。如果不足，先补覆盖面缺口，再谈视觉润色。
