# 常见失效模式

## 原则层失效

- 页面看起来完整，但不知道先做什么
- 只有信息，没有处理路径
- 异常、边界和恢复链路缺席

## 样式层失效

- 后台被做成营销页或展示页
- 主色、圆角、阴影使用过重
- 页面头过大，主内容被稀释

## 模式层失效

- 页面范式判断错位
- 主模式块不清，所有区块同权
- 业务组件名变多，但结构并没有新意

## 契约层失效

- 新组件只有名字，没有最小输入和状态
- 组件过于场景化，无法跨任务复用
- 动作入口缺失，组件只剩展示

## 可视化层失效

- 图表存在，但不支持判断或动作
- 单个当前值也被画成图
- 本应在表格里表达的状态被强行抽成图表

## 动效层失效

- 动效在制造存在感，而不是解释变化
- 装饰性动画过多，影响扫读和连续处理
- 去掉动画后，状态就无法理解
