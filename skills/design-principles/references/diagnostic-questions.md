# 诊断问题

当需求还模糊时，先问：

1. 这个页面服务的是哪一类工作任务？
2. 谁会使用这页？进入页面后是先判断局势，还是先处理对象？
3. 页面最重要的动作是什么？
4. 最怕出什么错？哪类异常最应该被提前暴露？
5. 哪些信息只是背景，不该占据主层级？
6. 这个页面和前后页面的工作流如何衔接？
7. 哪些信息可以量化？数量、范围、时限、阈值里哪些必须先写清？

## 使用方式

- 如果第 1、2、3 题答不清，先不要进入页面结构
- 如果第 4 题没有答案，说明异常链路还没被纳入设计
- 如果第 5、6 题没有答案，通常说明页面会走向信息同权和工作流断裂
- 如果第 7 题没有答案，提示和摘要大概率会滑向纯定性表达
