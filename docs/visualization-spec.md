# AI 管理后台可视化规范

这份文档对应“数据表达层”。后台的可视化目标不是炫技，而是支持判断和行动。

## 总原则

- 先信息，再图形
- 先比较，再装饰
- 先决策，再展示
- 能先给数值和范围，就不先给模糊判断

## 适合图形化的场景

- 趋势变化
- 类别 / 区域 / 角色对比
- 容量、负载或处理能力分布
- 目标达成与阈值关系

## 不适合图形化的场景

- 只有一两个数值，没有比较关系
- 需要立即处理单个对象
- 数据量很小，用文字更快理解

## 可视化组件建议

### 指标卡

- 用于首屏概览
- 必须配趋势或解释，不只放数字

### 进度 / 状态条

- 用于流程、SLA、覆盖率和阈值关系
- 适合后台高频判断

### 对比图

- 适合类别、区域、角色或队列横向比较
- 应优先支持排序和高亮异常项

### 表格中的可视信号

- 用标签、进度、语义色表达风险和状态
- 比单独再开图表更高效时，应优先使用

### 表格与浏览惯性

- 表格优先保持行列一致性，不为适配窄宽度牺牲阅读惯性
- 关键识别列、状态列、动作列应尽量保持单行扫读
- 空间不足时优先横向滚动，其次再考虑截断，不优先逐字换行
- 如果表格已经承担处理任务，就不要再把同一信息拆成卡片瀑布流

## AI 使用规则

- 如果图形不能直接支持决策，就不用图
- 如果一张图需要长说明，通常应拆成指标卡或表格
- 可视化必须服务于动作入口，而不是孤立存在
- 图形前能先给数值和范围，就不要只给模糊判断
