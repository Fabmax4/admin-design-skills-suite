# AI 管理后台样式规范

这份文档对应“约束层”。它不负责决定页面做什么，而是限制 AI 不能把后台做偏。

## 色彩

- 品牌主色：`#fa8c16`
- 成功色：沿用 `Ant Design` 绿色语义
- 警告色：沿用橙 / 金色语义
- 错误色：沿用红色语义
- 信息色：沿用蓝色语义

规则：

- 主色只用于关键按钮、重点状态、少量强调，不做大面积底色。
- 语义色只表达状态，不承担装饰任务。
- 默认信息卡、说明文字、辅助标签保持中性；`brand / info` 不自动获得彩色强调。
- 只有状态、风险、成功结果、一级动作这类确实需要视觉区隔的信息，才允许用语义色。
- 中性色优先承担版面结构、层级和阅读任务。
- 严禁把渐变当成后台的主要表面语言；除非有极强理由，否则默认不用渐变。

## 圆角

- 允许档位：`6 / 8 / 12 / 16`
- 默认信息卡、表单、表格容器优先使用 `8` 或 `12`
- 不再使用大圆角制造“卡片感”

规则：

- 后台骨架优先稳定，不强调展示型圆润风格。
- 同一页面内圆角种类不宜超过两档主档位。

## 密度

- 左侧导航展开宽度：`176px`
- 后台布局以紧凑、可扫读为目标
- 表单、表格、卡片的内边距优先统一，而不是每块单独发明节奏

规则：

- 通过排版和分组控制密度，不通过缩小一切字号来解决拥挤。
- 不要用过大的头部和留白把后台做成营销页。

## 间距

- 允许档位：`4 / 8 / 12 / 16 / 24 / 32 / 40 / 48`（对齐 `ant-design` 的 `sizeXXS / XS / SM / MD / LG / XL / XXL` 节奏）
- 表单控件行内最小间距 `8`
- 区块内小元素间距优先 `12-16`
- 区块与区块之间 `16-24`
- 页面主区块之间 `24-32`

规则：

- 不自造 `11px`、`18px`、`22px` 这类中间档。
- 强调层级不靠间距翻倍制造"呼吸感"，先靠信息优先级。
- 紧凑列表（表格、日志、对账行）可下沉到 `4 / 8` 档，但上下边距仍遵守节奏。

## 字号

- 正文：`14px`（对齐 `ant-design` 默认 `fontSize`）
- 次级、辅助文字：`12px`
- 标题档位：`16 / 20 / 24`（对应 `ant-design` 的 `h5 / h4 / h3`；`28 / 30` 仅用于登录 / 引导等极少数场景）
- 数字、金额等需要连续扫读的字段允许使用 `Tabular Nums`

规则：

- 不使用 `13 / 15 / 17` 这类非档位字号。
- 不通过放大字号制造强调，优先用语义色或字重。
- 不让正文比 `14px` 更小；`12px` 只留给辅助 / 标签 / 时间戳。

## 行高

- 正文：`1.5715`（对齐 `ant-design` 默认 `lineHeight`）
- 紧凑表格、日志列表：`1.5`
- 标题：`1.35`

规则：

- 不使用 `1.2` 以下的行高压缩正文。
- 表格单元格最小行高 `32px`，中文 + 英文混排不允许逐字换行。
- 标题和段落之间的节奏靠间距而不是压缩行高。

## 版式

- 优先使用稳定的左右 / 上下分区结构
- 主内容区应围绕单一任务组织
- 侧栏、页面头、主内容、辅助信息区层级要清晰
- 同类页面的导航分组、进入方式、返回方式应保持稳定，符合用户肌肉记忆

规则：

- 不随意打破 Ant Design 式后台骨架。
- 页面头只承载标题、状态摘要、一级动作，不堆长文。

## 边框与阴影

- 优先用边框、背景层次、分区留白组织页面
- 阴影保持弱化，只作为层级补充

规则：

- 不把阴影作为主要分组手段。
- 信息区块先靠结构区分，再靠视觉修饰。
- 不用渐变制造“高级感”或“完成度”，后台优先靠纯色、边框、留白和字体层级组织信息。
- 不用颜色去弥补信息表达模糊；提示和提醒优先先写清数量、时限和范围。

## 可访问性

- 保持键盘可达
- 主内容可聚焦
- 页面切换状态可播报
- 支持 `prefers-reduced-motion`

## 适合作为第二层 Skill 的原因

当页面目标已经明确，但视觉还未收敛时，AI 应调用样式规范层，把设计控制在后台语境内。
