# AI 管理后台样式规范

这份文档对应“约束层”。它不负责决定页面做什么，而是限制 AI 不能把后台做偏。

## 色彩

- 品牌主色：`#fa8c16`
- 成功色：沿用 `Ant Design` 绿色语义
- 警告色：沿用橙 / 金色语义
- 错误色：沿用红色语义
- 信息色：沿用蓝色语义

规则：

- 主色只用于关键按钮、重点状态、少量强调，不做大面积底色。
- 语义色只表达状态，不承担装饰任务。
- 默认信息卡、说明文字、辅助标签保持中性；`brand / info` 不自动获得彩色强调。
- 只有状态、风险、成功结果、一级动作这类确实需要视觉区隔的信息，才允许用语义色。
- 中性色优先承担版面结构、层级和阅读任务。
- 严禁把渐变当成后台的主要表面语言；除非有极强理由，否则默认不用渐变。

## 圆角

- 允许档位：`6 / 8 / 12 / 16`
- 默认信息卡、表单、表格容器优先使用 `8` 或 `12`
- 不再使用大圆角制造“卡片感”

规则：

- 后台骨架优先稳定，不强调展示型圆润风格。
- 同一页面内圆角种类不宜超过两档主档位。

## 密度

- 左侧导航展开宽度：`176px`
- 后台布局以紧凑、可扫读为目标
- 表单、表格、卡片的内边距优先统一，而不是每块单独发明节奏

规则：

- 通过排版和分组控制密度，不通过缩小一切字号来解决拥挤。
- 不要用过大的头部和留白把后台做成营销页。

## 版式

- 优先使用稳定的左右 / 上下分区结构
- 主内容区应围绕单一任务组织
- 侧栏、页面头、主内容、辅助信息区层级要清晰
- 同类页面的导航分组、进入方式、返回方式应保持稳定，符合用户肌肉记忆

规则：

- 不随意打破 Ant Design 式后台骨架。
- 页面头只承载标题、状态摘要、一级动作，不堆长文。

## 边框与阴影

- 优先用边框、背景层次、分区留白组织页面
- 阴影保持弱化，只作为层级补充

规则：

- 不把阴影作为主要分组手段。
- 信息区块先靠结构区分，再靠视觉修饰。
- 不用渐变制造“高级感”或“完成度”，后台优先靠纯色、边框、留白和字体层级组织信息。
- 不用颜色去弥补信息表达模糊；提示和提醒优先先写清数量、时限和范围。
- 页面级 `Selection Action Bar`（选择操作栏）优先使用白底、顶部边线和弱上投影；不要做成普通卡片，也不要覆盖表格行、空态、分页或横向滚动控件。
- `Selection Action Bar` 里的金额、数量和范围默认保持中性，只有风险、欠额、阻断条件或危险动作才使用语义色。

## 可访问性

- 保持键盘可达
- 主内容可聚焦
- 页面切换状态可播报
- 支持 `prefers-reduced-motion`

## 适合作为第二层 Skill 的原因

当页面目标已经明确，但视觉还未收敛时，AI 应调用样式规范层，把设计控制在后台语境内。
