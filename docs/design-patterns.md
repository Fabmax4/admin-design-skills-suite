# AI 管理后台设计模式

这份文档对应“结构层”。AI 在这一步不关心具体业务名，而是先选择页面范式和模式块。规则层应尽量保持行业中立，具体行业语境留在案例层和提示层。

## 页面范式

### 工作台

- 目标：快速理解全局状态并进入高频动作
- 常见模式：`Summary Metrics`、`Signal Panel`、`Queue Block`

### 查询列表

- 目标：查找、筛选、比较、批量处理对象
- 常见模式：筛选区、批量操作区、表格、摘要侧栏

### 详情页

- 目标：在稳定上下文中查看对象状态、规则和关联信息
- 常见模式：`Snapshot Block`、`Rule Block`、信号区

### 编辑页

- 目标：维护单个对象内容
- 常见模式：`Section Editor`、`Journey Editor`、`List Editor`

### 配置页

- 目标：定义流程、策略、发布规则
- 常见模式：步骤编排、规则块、预检区

### 异常处理台

- 目标：发现、分桶、派发、恢复异常
- 常见模式：`Queue Block`、`Rule Block`、`Signal Panel`

### 资源运营台

- 目标：调配资源、容量、队列和处理能力
- 常见模式：`Signal Panel`、计划区、影响范围区

### 协同台

- 目标：协调多个角色、地点、班次或责任链上的接手动作
- 常见模式：`Queue Block`、`Status Rail`、`Rule Block`

## 模式块

### Summary Metrics

- 用于建立局势概览
- 必须同时有数值、趋势、解释语境

### Status Rail

- 用于表达阶段型流程
- 必须体现阶段、当前状态、时效或进度

### Signal Panel

- 用于表达健康度、风险和资源压力
- 必须体现当前状态、阈值或占比、建议动作

### Queue Block

- 用于承接等待处理的对象
- 必须体现责任、时限、优先级、动作

### Rule Block

- 用于表达限制、规则、预案和发布控制
- 必须支持扫读，不写成长段说明书

### Snapshot Block

- 用于在不切走上下文时提供对象摘要
- 适合详情侧栏、列表抽屉、联动预览

### Section Editor

- 用于非线性编辑
- 一个区块只负责一类内容

### Journey Editor

- 用于步骤型内容维护
- 适合流程路径、处理步骤、责任交接链

### List Editor

- 用于重复结构的维护
- 适合规则、卖点、FAQ、步骤清单

## 临时任务与浮层选择

### Dialog / Modal

- 用于阻断式确认、短时决策、不可逆操作确认
- 只适合短内容和清晰主次操作，不适合长表单和长滚动

### Drawer / Side Panel

- 用于在不离开当前页面的前提下查看详情、编辑子对象、处理临时任务
- 适合需要保留父页面上下文的场景

### Popover

- 用于锚定在触发器上的短说明或轻量动作
- 只承载很短的内容和少量动作，不承载主任务

### Tooltip

- 用于单句解释、标签释义、图标含义补充
- 只负责解释，不负责承载状态判断和操作

### Toast Feedback

- 用于操作完成后的非阻断反馈
- 轻反馈用 `Message`，复杂异步反馈或系统主动推送用 `Notification`

## 浮层选择顺序

1. 需要用户先做决定，且任务短而阻断：`Dialog / Modal`
2. 需要保留父页面上下文并承接临时查看或编辑：`Drawer / Side Panel`
3. 需要锚定触发器展示短内容或轻动作：`Popover`
4. 只需要一句解释：`Tooltip`
5. 不需要用户立即决策，只需要反馈结果：`Message / Notification`

## 列表高频交互

### 搜索区

- 用于快速定位单个对象、编号、人员或位置
- 应优先放在筛选区最前，保持用户先“找对象”再“缩范围”的惯性
- 搜索只负责快速定位，不替代结构化筛选

### 筛选区

- 用于按状态、类型、来源、时间、区域等维度缩小结果范围
- 高频筛选固定在首行，低频筛选折叠或后置
- 重置动作必须清晰，不能让筛选条件变成隐性状态

### 分页区

- 用于结果量过大时保持扫描节奏稳定
- 必须让用户明确当前范围、总量和翻页位置
- 翻页不应破坏现有筛选、搜索和列配置状态

### 列配置

- 用于列数超过 `15`、角色扫读重点不同、但核心结构仍需稳定的表格
- 必须锁定主键列和操作列，只允许隐藏次要列
- 优先通过显示列和列顺序管理密度，不先破坏行列阅读惯性

### 提示与播报

- 用于提醒、摘要、风险提示和结果播报
- 能定量先定量，不能定量再定性
- 至少回答一类问题：影响多少、还剩多久、波及范围、下一步动作

## 信息架构与导航

- 同类对象应保持同类寻址方式，不因为单页优化打破全局规律。
- 导航分组、命名、返回路径、详情进入方式应前后一致。
- 页面之间的承接关系应稳定，符合用户肌肉记忆，而不是每页重新学习。

### 批量操作

- 用于多条对象共享同一处理动作的列表页
- 必须明确已选数量、动作影响范围和危险等级
- 危险批量操作必须二次确认，轻量动作不另开新容器

### Selection Action Bar（选择操作栏）

- 用于表格多选后触发页面级主动作，例如批量确认、导出、结算、开票或复核。
- 通用识别名为 `Selection Action Bar`；可识别别名包括 `Selection Summary Bar`、`Batch Action Bar`、`Bulk Action Bar`、`Selected Rows Toolbar`、选择操作栏、选择汇总操作栏、已选预览栏。
- 必须同时说明已选数量、作用范围、跨页选择语义、关键指标和主动作禁用原因。
- 它是页面级操作栏，不是普通信息卡；贴底时必须给表格、空态、分页和横向滚动控件预留安全空间。

### 详情抽屉

- 用于在不离开列表的前提下查看详情、补充判断、执行临时动作
- 必须包含对象标题、摘要、当前状态和主动作
- 若存在未保存内容，关闭前必须确认并回到来源

## 模式选择顺序

1. 先定页面范式
2. 再定主模式块
3. 再定辅助模式块
4. 最后决定是否需要业务语义包装

## 不该做的事

- 先命名业务组件，再倒推页面结构
- 用一个新卡片名掩盖其实是旧模式的事实
- 为了“看起来丰富”把所有模式都塞进同一页
- 把长表单、长详情或关键决策都塞进 `Modal`
- 把必要信息和关键操作藏进 `Tooltip` 或 `Popover`
- 把搜索、筛选、分页拆成互不相关的零散区域，破坏查询列表的浏览节奏
- 让导航、返回路径、详情进入方式在同类页面里来回变化，破坏用户肌肉记忆
- 允许用户隐藏主键列、打乱主操作列，破坏列表稳定结构
- 在未选中对象时仍然暴露可误触的批量危险动作
- 把 `Selection Action Bar` 做成表单里的普通卡片，或让它覆盖表格、空态、分页和横向滚动控件
- 把完整主流程塞进详情抽屉，导致父列表和返回语义失效
