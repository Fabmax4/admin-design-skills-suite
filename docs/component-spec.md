# AI 管理后台组件规范

这份文档对应“组件契约层”。这里的组件不是具体 Vue 文件，而是 AI 在输出页面时应遵守的抽象组件协议。

## 组件规范的目标

- 让 AI 先复用契约，再决定是否需要新组件
- 让组件命名来自模式，而不是来自场景
- 让业务组件只是抽象契约的语义包装

## 抽象组件契约

### Summary Metrics

- 最小输入：
  数值、趋势、说明、动作入口
- 必备状态：
  正常、上升、下降、风险
- 禁止：
  只有数字没有解释

### Signal Panel

- 最小输入：
  当前状态、阈值、影响范围、建议动作
- 必备状态：
  健康、关注、告警
- 禁止：
  只报告现状，不给判断

### Queue Block

- 最小输入：
  对象名、责任人、优先级、时限、动作
- 必备状态：
  待处理、处理中、已升级、已恢复
- 禁止：
  只有列表没有操作

### Scan Table

- 最小输入：
  稳定列定义、行对象、主键、主要状态信号、动作列
- 必备状态：
  列顺序稳定、列宽有策略、窄视口可横向滚动、关键字段保持单行扫读、单元格尺寸不随机波动
- 禁止：
  任由表格挤压成逐字换行，破坏行列阅读惯性

### Search Bar

- 最小输入：
  关键词范围、占位提示、触发动作、清空动作
- 必备状态：
  默认、输入中、已清空、定位结果
- 禁止：
  用搜索去替代结构化筛选，或让用户不知道搜索作用于什么范围

### Filter Panel

- 最小输入：
  筛选项、默认值、重置动作、结果范围说明
- 必备状态：
  默认、部分筛选中、已重置、结果收窄
- 禁止：
  让筛选条件隐形存在，或把高频筛选藏到低可见位置

### Pagination Bar

- 最小输入：
  当前页、总量、每页数量、翻页动作
- 必备状态：
  首页、中间页、末页、空结果
- 禁止：
  翻页后丢失搜索、筛选和列配置上下文

### Column Configurator

- 最小输入：
  列定义、锁定列、默认顺序、可见状态、重置动作
- 必备状态：
  默认、部分隐藏、重排后稳定、可恢复默认
- 使用边界：
  当列数超过 `15` 时必须提供；低于该阈值时优先先靠列宽、冻结列和横向滚动解决复杂度
- 禁止：
  允许隐藏主键列或操作列，或让列设置破坏浏览阅读惯性

### Prompt / Notice Block

- 最小输入：
  事实、影响范围、时限或阈值、下一步动作
- 必备状态：
  默认、提醒、风险、结果反馈
- 禁止：
  只有“尽快处理”“风险较高”这类纯定性表述，没有数量、时限或范围

### Bulk Action Bar

- 最小输入：
  已选数量、动作列表、风险级别、清空选择动作
- 必备状态：
  未选中、已选中、执行中、危险确认
- 禁止：
  未选中时暴露可误触的危险动作，或让批量动作语义与单行动作冲突

### Rule Block

- 最小输入：
  规则标签、规则内容、适用范围
- 必备状态：
  默认、警告、限制
- 禁止：
  长段堆字

### Snapshot Block

- 最小输入：
  对象标题、关键状态、补充字段
- 必备状态：
  简洁、稳定、可快速扫读
- 禁止：
  把快照做成完整详情页

### Detail Drawer

- 最小输入：
  对象标题、摘要、主任务、关闭语义、返回来源
- 必备状态：
  打开、加载中、编辑中、待保存、已回到来源
- 禁止：
  把完整主流程塞进抽屉，或在有未保存内容时直接丢失上下文

### Section Editor

- 最小输入：
  区块标题、区块说明、区块内容、保存语义
- 必备状态：
  编辑中、已更新、待校验
- 禁止：
  把整个编辑页做成一个巨型表单

### Journey Editor

- 最小输入：
  步骤名、责任方、时效、说明
- 必备状态：
  顺序明确、步骤可读
- 禁止：
  只有步骤名，没有责任和结果

### List Editor

- 最小输入：
  项标题、标签或类型、描述
- 必备状态：
  可增删、可排序、可快速浏览
- 禁止：
  失去清单结构，退化成普通段落

## 浮层与反馈组件契约

### Dialog / Modal

- 最小输入：
  标题、触发原因、主操作、取消操作、关闭语义
- 必备状态：
  打开、确认中、关闭、不可逆风险
- 关闭规则：
  普通确认可允许 `Esc` 或点击遮罩关闭；高风险操作默认不允许误触关闭
- 禁止：
  长表单、长详情、长滚动、多层嵌套

### Drawer / Side Panel

- 最小输入：
  触发对象、上下文摘要、主任务、关闭或返回动作
- 必备状态：
  预览、编辑中、加载中、待保存
- 关闭规则：
  必须能回到触发来源；有未保存内容时必须给出关闭确认
- 禁止：
  承载危险确认、连续套娃式多层抽屉、把整页信息强塞进侧栏

### Popover

- 最小输入：
  触发器、锚点内容、关闭条件
- 必备状态：
  关闭、打开、禁用
- 内容上限：
  短说明或 `1-3` 个轻动作
- 禁止：
  长表单、长文、关键流程、关键确认

### Tooltip

- 最小输入：
  目标元素、单句解释
- 必备状态：
  默认、悬停或聚焦显示、禁用
- 内容上限：
  一句短解释，不承载链接、按钮或段落
- 禁止：
  把必需信息只放在 Tooltip 里

### Message / Toast

- 最小输入：
  反馈级别、短文案、持续时间
- 必备状态：
  成功、信息、警告、错误
- 使用边界：
  只反馈结果，不要求用户在 Toast 内做主决策
- 禁止：
  塞入复杂描述、长操作链或关键确认

### Notification

- 最小输入：
  标题、描述、来源、可选动作
- 必备状态：
  信息、异步完成、警告、异常升级
- 使用边界：
  用于较复杂反馈、异步任务完成、系统主动推送
- 禁止：
  高频刷屏，或替代列表/详情页承载主体信息

## 前端运行时规则

- 统一入口：
  页面和业务组件里的程序化反馈统一走项目内的反馈封装，不直接散落 `message`、`notification`、`Modal.confirm` 或 `App.useApp()`。
- 上下文要求：
  根布局必须包裹 `a-app`，保证反馈容器和上下文一致。
- 风险确认：
  高风险操作默认走 `confirmRisk`，包含明确标题、主次按钮和误触关闭保护。
- 轻反馈：
  只反馈短结果时使用 `info / success / warning / error`。
- 复杂反馈：
  异步结果、系统推送、需要标题和补充描述时使用 `notifyInfo / notifySuccess / notifyWarning / notifyError`。
- 禁止：
  一个页面自己维护一套提示默认值、按钮文案和关闭规则。

## 通用信息表达规则

- 提示类信息能定量先定量，不能定量再定性。
- 能写清数量、比例、时限、范围、阈值时，不只写情绪化判断。
- 摘要区和提示区必须帮助用户降低不确定性，而不是重复字段或堆形容词。

## 新组件判定标准

只有同时满足以下条件，才值得新增组件：

1. 现有模式块确实无法表达
2. 输入契约和交互结构稳定
3. 可以跨多个后台场景复用
4. 不是仅仅换了业务文案

如果不满足，就应该继续复用现有模式。
